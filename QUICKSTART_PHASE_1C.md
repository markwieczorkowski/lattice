# Quick Start - Phase 1C Component Configuration

## 🎯 What's New in Phase 1C

Phase 1C adds **component configuration and management** with menu system, dialogs, and live preview.

## 🎨 Configure Components

### Open Component Menu

1. **Find the menu button** in the upper-right corner of any component
2. **Click the three-line icon** (≡)
3. Menu drops down with options

### Configure Settings

1. Click menu → **Configure**
2. Configuration dialog opens
3. Adjust settings (changes apply immediately):
   - **Background Color** - Click color picker
   - **Background Opacity** - Drag slider (0-100%)
   - **Title Text** - Type new title
   - **Text Color** - Click color picker
4. **Watch live preview** as you make changes
5. **Click outside or X** to close dialog

### Remove Component

1. Click menu → **Remove**
2. Confirmation dialog appears
3. Click **OK** to delete (or Cancel to keep)
4. Component is permanently removed

## ⚙️ Component Menu Options

Every component has:
- ⚙️ **Configure** - Open settings dialog
- 🗑️ **Remove** - Delete component (with confirmation)

## 🎛️ Configuration Controls

### Color Pickers
- Click to open color selector
- Choose color from palette
- Hex value shown next to picker
- Changes apply instantly

### Opacity Slider
- Drag slider left/right
- 0% = fully transparent
- 100% = fully opaque
- Percentage shown above slider

### Text Input
- Click to focus
- Type new title
- Updates as you type
- Supports emojis and special characters

## ⌨️ Keyboard Shortcuts

- **ESC** - Close dialog or cancel removal

## 💡 Tips

1. **Live Preview** - See changes immediately without saving
2. **Experiment** - Try different colors and opacities
3. **Transparency** - Lower opacity to see grid through component
4. **Emojis** - Add emojis to titles for visual organization
5. **High Contrast** - Use complementary colors for readability
6. **Multiple Configs** - Each component remembers its own settings
7. **Undo Remove** - There's no undo! Be careful with Remove

## 🎨 Color Suggestions

**Light Themes:**
- Background: Light color + low opacity (20-40%)
- Text: Dark color for contrast

**Dark Themes:**
- Background: Dark color + medium opacity (50-70%)
- Text: White or light color

**Vibrant Themes:**
- Background: Bright color + medium opacity (40-60%)
- Text: White for readability

## 📋 Workflow Examples

### Organize by Category

1. Place multiple test components
2. Configure each with category color:
   - 🔴 Red: Urgent/Important
   - 🟢 Green: Completed/Active
   - 🔵 Blue: In Progress
   - 🟡 Yellow: Waiting/Pending
3. Adjust titles to match purpose
4. Arrange spatially on board

### Create Visual Hierarchy

1. Important items: High opacity (80-100%)
2. Secondary items: Medium opacity (50-60%)
3. Background items: Low opacity (20-30%)
4. Use size and color to emphasize

### Experiment with Styles

1. Open config dialog
2. Drag opacity slider back and forth
3. Try different color combinations
4. Find what works for your use case
5. Close dialog when satisfied

## 🔧 Troubleshooting

**Menu button not visible?**
- Hover over upper-right corner of component
- White button with three lines should appear

**Changes not saving?**
- Changes save immediately to app state
- No "Save" button needed
- Persistence to localStorage coming in Phase 1E

**Dialog won't close?**
- Click X button in header
- Click outside dialog area
- Press ESC key

**Accidentally removed component?**
- No undo available
- Create new component and reconfigure
- Full undo/redo coming in future phase
